## 一、 信息获取与整理

### 1. 信息抓取的三种方式

在此课程中，老师提到了三种主要的信息抓取手段：

1. **Python 脚本**：编写代码进行定制化抓取。
    
2. **n8n 自动化**：用于提取微信公众号等平台的数据。
    
3. **RSS 订阅**：利用 RSS 链接获取信息流。
    

### 2. 文件整理的注意事项

- **权限风险**：在使用 AI 工具辅助整理文件时，需注意权限控制。如果赋予 AI 过高权限，可能会导致重要文件被误删。
    
- **最佳实践**：建议使用 Python 编写专门的脚本来处理文件整理任务，而非直接交给 AI 自由操作。

## 二、 AI 产品全流程开发详解

产品开发不是一蹴而就的，而是一个循环迭代的过程：**需求 -> Demo -> 可行性 -> 规划 -> PRD -> UI -> 开发 -> 测试 -> 上线 -> (迭代优化) -> 回到 PRD**。

### 第一阶段：明确需求

- **核心问题**：明确产品的用户是谁？用户为什么要用这个产品？
    
- **迭代认知**：需求并不是一开始就是完美或成功的，需要在不断的实践和版本迭代中修整，最终形成明确的方向。
    

### 第二阶段：Demo 验证与可行性分析

- **快速验证**：制作快速 Demo 进行验证。
    
- **AI 沟通技巧**：
    
    - **对齐需求**：在使用 AI（如 Claude Code）时，先告知要做什么，让 AI 复述它的理解，再进行纠正，直到双方理解一致。
        
    - **模型对比**：建议同时与几个聪明的模型（如 Codes 和 Claude Code）沟通，对比它们的理解，以此来对齐需求的颗粒度。
        

### 第三阶段：产品版本规划

- **分步实施**：不要试图在一个版本中实现所有功能。
    
- **明确界限**：规划清楚每一个版本具体要实现什么，制定清晰的路线图（从 V1.0 到 V20.0）。
    

### 第四阶段：撰写 PRD（产品需求文档）

> **核心思考**：开发一个产品，**写 PRD 的时间可能比 AI 开发的时间长得多**。PRD 写得越清楚，AI 开发的时间就越短。

- **撰写方式**：
    
    1. **提示词 (Prompt)**：使用特定的提示词模板撰写（需下载模板）。
        
    2. **Skills (技能)**：Skills 文件，让 Claude Code 加载后直接生成 PRD。
        
- **工具搭配建议**：推荐使用 **Gemini 3** 来撰写 PRD，然后用 **Claude Code** 进行后续开发。
    

### 第五阶段：UI 设计与开发

- **开发策略**：
    
    1. **生成计划**：让 AI 先写一份详细的开发计划。如果不懂技术，可以严格按照 AI 的计划一步步执行。
        
    2. **分节点开发**：不要让 AI 一次性生成所有代码，而是分节点、分步骤地进行开发。
        
- **概念补充**：在此阶段可能会用到 **ASCII 图**（字符画）来辅助表达或设计。
- **为什么 Vibe Coding 要用 ASCII 图？**
	- 因为 LLM (大模型) 是基于文本的，它看不懂 JPG 图片。
	- 但是它能完美理解用字符画出来的流程图（ASCII Art）。
	- **技巧：** 在写 PRD 时，用 ASCII 画出数据库结构或页面跳转逻辑，AI 生成代码的准确率会提升。

### 第六阶段：Debug（找问题）

- **核心思路**：人不再是亲自找问题的那个人，而是“找问题的导演”**。
    
- **执行方式**：通过打日志（Log）等方式，指导 AI 去哪个方向排查问题。

## 三、 核心概念与技巧辨析

### 1. 提示词 (Prompt) vs Skills (技能)
[[Skills (技能) 深度解析]]
- **共同点**：两者有很多重复概念。
    
- **区别**：
    
    - **Skills 是“大号的提示词”**。
        
    - **调用方式**：提示词通常是全量输入（如每次都要打 1000 字）；而 Skills 支持**按需调用**（Skills 文件可以有几万字，但每次使用时只需调用其中相关的 1000 字），使用体验更舒适。
        

### 2. IDE 与终端工具的关系

- **误区澄清**：在 IDE（如 Cursor 或其它编辑器）中使用 Codes 或 Claude Code 时，IDE 仅仅是提供了一个**展示终端环境**的功能。
    
- **本质**：它们之间没有底层关联，并不是 Codes 插件“安装”进了 IDE，仅仅是 IDE 加载了终端而已。
    

### 3. 实用快捷键

- **恢复对话**：在 Claude Code 和 Codes 中，回到上一轮对话的快捷键通常是 `/resume`。
