# 前后端开发学习笔记

## 1. 核心概念：餐厅比喻

把开发一个互联网产品（网站/APP）想象成**“开一家餐厅”**。

|**角色**|**技术概念**|**职责**|**形象比喻**|
|---|---|---|---|
|**前厅**|**前端 (Frontend)**|负责展示、交互、颜值|装修队、服务员|
|**后厨**|**后端 (Backend)**|负责逻辑、计算、处理数据|大厨|
|**服务员**|**API (接口)**|负责传话、传递数据|端盘子的人|
|**仓库/账本**|**数据库 (Database)**|负责存储数据、永久记录|保险柜、Excel|

---

## 2. 前端 (Frontend)：餐厅的前厅

**核心任务：** 给用户看，给用户用。负责貌美如花。

### 前端“三剑客” (技术栈)

不管网页多复杂，本质上都是这三样东西在工作：

1. **HTML (结构) —— 毛坯房 / 骨架**
    
    - **作用：** 把东西摆出来（比如：这里放个标题，那里放张图）。
        
    - **如果没有它：** 网页就是一张白纸。
        
2. **CSS (样式) —— 精装修 / 皮囊**
    
    - **作用：** 负责美颜（比如：字体变大，按钮变红，排版整齐）。
        
    - **如果没有它：** 网页就像没装修的毛坯房，丑到没法看。
        
3. **JavaScript (JS) —— 魔法师 / 肌肉**
    
    - **作用：** 负责动作和交互（比如：点击按钮弹窗，下拉刷新）。
        
    - **如果没有它：** 网页就是个死板的画报，点不动。
        

---

## 3. 后端 (Backend)：餐厅的后厨

**核心任务：** 处理业务逻辑，算账，存数据。

### 后端核心组件

1. **服务器 (Server)：** 厨房本身。一台 24 小时开机的电脑，提供场地。
    
2. **后端语言 (厨师)：**
    
    - **Python：** 金融/数据分析首选。聪明，擅长数学和逻辑。
        
    - **Java：** 大企业首选。严谨、稳定，适合银行系统。
        
    - **Node.js：** 前端 JS 语言的后端版，适合快速开发。
        
3. **数据库 (数据库)：** 仓库 / 记账本。
    
    - **MySQL：** 最常用的数据库软件。
        
    - **本质：** 一个**永不关机、超大容量、支持多人同时操作的 Excel 表格**。
        
    - **作用：** 厨师（代码）只负责算，算完的数据必须锁进这里，否则断电就丢。
        

---

## 4. 它们怎么沟通？

前端（手机）和后端（服务器）之间不能直接说话，必须按规则来。

- **连接方式：** **API (接口)**。前端通过 API 喊后端干活。
    
- **沟通语言：** **JSON**。
    
    - 一种标准的数据格式，像**“填空题”**一样的纸条。
        
    - _例子：_ `{ "账号": "张三", "余额": "100" }`
