这份笔记汇总了关于“智能体（Agent）”的分类逻辑及其底层大模型能力的差异

---

### 核心定义

**智能体 (Agent) = 大模型 (Brain) + 感知 (Sensory) + 工具 (Tools/Action)**

区别于单纯的 Chat（聊天），智能体的核心在于它不仅能“说”，还能“做”（调用搜索、运行代码、操作软件）。

---

### 一、 三大主流形态对比

我们将智能体市场分为三个层级，分别对应不同的使用场景和控制权：

#### 1. 低代码开发平台 (Low-Code Agent Platforms)

- **代表产品：** Coze (扣子), Dify, LangFlow
    
- **核心类比：** **兵工厂 / 全屋定制工作台**
    
- **本质：** 生产工具。它本身不是智能体，而是**让你用来“造”智能体**的环境。
    
- **控制权：** **100% 在搭建者手中**。你需要设计每一步流程（SOP）。
    
- **你的场景：** 搭建企业内部固化的业务流（如：合规审查 Bot、自动写周报助手）。
    

#### 2. 通用智能体 (General-Purpose Agents)

- **代表产品：** Manus, AutoGPT, OpenAI Operator
    
- **核心类比：** **全能实习生 / 盲盒管家**
    
- **本质：** 黑盒劳动力。你只给一个模糊的目标，它自主拆解任务并执行。
    
- **控制权：** **在 AI 手中**。过程不可见，主打“结果导向”。
    
- **你的场景：** 个人提效或非严肃场景的探索（如：全网调研某家信托公司的舆情）。
    

#### 3. 垂类智能体软件 (Vertical Agent Software)

- **代表产品：** Cursor (编程), Jasper (营销), Wind/Bloomberg AI (金融)
    
- **核心类比：** **特种兵 / 资深老法师**
    
- **本质：** 封装好的成品软件。内置了特定领域的最佳实践（Best Practice）。
    
- **控制权：** **在软件厂商手中**。你只能使用其功能，但它在特定领域极强。
    
- **你的场景：** 专业生产力工具（如：用 Cursor 写 Vibe Coding 脚本，用 Wind 提取数据）。

### 二、 大模型（LLM）在其中的能力差异

虽然都是用大模型作为核心引擎，但输出的“能量形式”完全不同：

| **维度**       | **低代码平台 (Coze/Dify)**                                                                                                                  | **通用智能体 (Manus)**                                                                                                                      | **垂类软件 (Cursor)**                                                                                                                                       |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **LLM 扮演角色** | **调度员 & 翻译官**<br><br>  <br><br>(Router & Processor)                                                                                    | **项目经理 & 推理者**<br><br>  <br><br>(Planner & Reasoner)                                                                                   | **领域专家 & 老师傅**<br><br>  <br><br>(Domain Expert)                                                                                                         |
| **核心能力体现**   | **1. 意图识别：** 精准判断用户是想查天气还是写代码。<br><br>  <br><br>**2. 参数提取：** 从自然语言中提取 `600519` 等关键参数传给插件。<br><br>  <br><br>**3. 遵从性：** 严格按你画的流程图走，不加戏。 | **1. 任务拆解 (CoT)：** 自主把“大目标”拆成“任务链”。<br><br>  <br><br>**2. 自我反思：** 路走不通时（如报错），会自己换条路试。<br><br>  <br><br>**3. 工具调用：** 自主决定何时打开浏览器，何时写文件。 | **1. 上下文理解：** 瞬间读懂几百个文件的关联（Project Context）。<br><br>  <br><br>**2. 微调输出：** 代码/文案符合行业规范（如 PEP8 或 金融合规语态）。<br><br>  <br><br>**3. 预判：** 猜到你下一步想做什么（如代码补全）。 |
| **智商侧重点**    | **理解力 + 格式化能力**                                                                                                                        | **逻辑推理力 + 规划力**                                                                                                                        | **记忆力 + 经验值**                                                                                                                                           |

### 三、 对“金融 AI 体系”搭建的启示


1. **左端（投研/数据清洗）：** 采用 **垂类软件思维 (Cursor)**。利用 LLM 的上下文能力，处理复杂的研报和代码，解决“Vibe Coding”的技术门槛。
    
2. **中间（业务流程/合规）：** 采用 **低代码平台 (Coze)**。利用 LLM 的遵从性，将“老法师”的业务逻辑固化为标准工作流，确保合规不越界。
    
3. **右端（客户洞察/KYC）：** 未来可探索 **通用智能体**。在确保隐私的前提下，利用 LLM 的推理力，从零散信息中拼凑客户的隐性需求画像。
