## 一、项目背景与目标

### 1.1 项目背景

作为服务于金融机构的专业研究机构，对信托行业的机构信息变动需要保持高度敏感性。目前主要通过人工方式关注多个信息源（公众号、新闻网站、监管平台等）来获取人事变动、股权变更等关键信息，存在以下痛点：

- **人力成本高**：需要专人持续关注多个信息渠道
- **信息遗漏风险大**：人工监控容易出现疏漏，导致关键信息延迟获取
- **数据更新不及时**：从信息发布到数据库更新存在较长时滞
- **信息提取效率低**：需要人工从新闻稿、公告中提取结构化信息

### 1.2 项目目标

通过自动化和AI技术实现信托机构信息的智能监控系统，具体目标包括：

1. **自动跟踪**：基于关注公司列表，自动监控多个信息源的更新
2. **智能提取**：利用AI自动提取人事变动、股权变更等结构化信息
3. **数据对比**：自动与现有数据库进行对比，识别变动
4. **及时预警**：1-2天内完成信息发现、提取、推送
5. **人工审核**：支持人工复核机制，确保数据准确性

## 二、业务需求分析

### 2.1 监控范围

| 监控对象 | 详细说明 |
|---------|---------|
| 机构类型 | 信托公司（优先），后期可扩展至银行、证券、基金等 |
| 人事层级 | 董事长、总经理、副总、部门总经理/总监及以上职位 |
| 股权信息 | 股东变更、持股比例变化、实际控制人变更 |
| 信息源 | 监管平台（银保监会）、公司官网、微信公众号、财经媒体 |

### 2.2 性能要求

| 指标项 | 目标值 |
|-------|--------|
| 时效性 | 信息发布后1-2天内完成提取和推送 |
| 准确率 | AI提取准确率≥85%（通过人工审核保障最终准确性） |
| 覆盖率 | 漏报率<10%（宁可多推送，保持较低漏报率） |
| 误报容忍度 | 可接受一定误报，通过人工筛选过滤 |

### 2.3 集成要求

- 与现有CRM系统中的关系型数据库集成
- 支持数据双向同步（读取现有数据、写入更新信息）
- 提供审核界面供研究人员确认和修正AI提取的信息

## 三、核心功能模块设计

系统架构采用模块化设计，分为五大核心模块：

### 3.1 数据采集层

**功能：**
- 定时抓取监管平台公告（银保监会网站）
- 监控信托公司官网动态
- 订阅微信公众号文章（通过第三方服务或RSS）
- 抓取财经媒体新闻（新浪财经、财新等）

**技术要点：**
- 网页爬虫（BeautifulSoup / Scrapy）
- RSS订阅解析
- 微信公众号抓取方案（需评估技术可行性）
- 定时任务调度（Cron / APScheduler）

### 3.2 信息提取层

**功能：**
- 识别文本中的关键信息（人物、职位、公司、时间）
- 提取人事变动信息（任命、离职、调动）
- 提取股权变动信息（股东变更、持股比例）
- 结构化输出（JSON格式）

**技术方案：**
- 大语言模型API（Claude / GPT-4）
- Prompt工程优化（设计专用提取模板）
- Few-shot学习（提供典型案例样本）

### 3.3 结构化存储层

**功能：**
- 存储原始抓取内容
- 存储AI提取的结构化信息
- 记录处理状态（待审核、已确认、已忽略）
- 保存信息来源和时间戳

**数据结构示例：**
```json
{
  "id": "20260206001",
  "company": "中信信托有限责任公司",
  "person_name": "张三",
  "change_type": "任命",
  "position": "信息科技部总经理",
  "effective_date": "2026-02-01",
  "source_url": "https://...",
  "source_type": "公司官网",
  "crawl_time": "2026-02-06 10:00:00",
  "extract_time": "2026-02-06 10:05:00",
  "confidence": 0.92,
  "status": "pending_review",
  "reviewer": null,
  "review_time": null
}
```

### 3.4 对比告警层

**功能：**
- 与CRM数据库中的现有记录对比
- 识别新增、变更、冲突信息
- 优先级排序（重要职位变动优先级更高）
- 生成变动报告

**对比逻辑：**
- 姓名+公司匹配：检查人员是否已存在
- 职位对比：识别职位变动
- 时间验证：确保信息为最新

### 3.5 人工审核层

**功能：**
- 展示待审核信息列表
- 提供原文链接查看
- 支持确认、修正、忽略操作
- 一键更新至CRM数据库
- 邮件/企业微信通知机制

### 4.2 方案A：Python轻量级方案（✅ 推荐）

#### 技术架构
- **编程语言**：Python 3.9+
- **爬虫框架**：Scrapy / BeautifulSoup + Requests
- **AI服务**：Claude API / OpenAI GPT-4 API
- **数据库**：SQLite（开发）/ PostgreSQL（生产）
- **任务调度**：APScheduler / Celery
- **Web框架**：Flask / FastAPI（审核界面）
- **部署**：Docker + Linux服务器

#### 核心优势
- ✅ **完全可控**：代码完全自主，可根据需求灵活调整
- ✅ **成本透明**：主要成本为AI API调用费，可精确估算
- ✅ **易于集成**：可直接连接CRM数据库，无需中间层
- ✅ **可扩展性强**：后期可轻松添加新数据源、新字段
- ✅ **技术生态成熟**：Python爬虫和AI开发资源丰富

#### 主要挑战
- ⚠️ **开发周期较长**：需2-3周完成MVP
- ⚠️ **需要技术人员**：后期维护需要Python开发能力
- ⚠️ **微信公众号抓取**：存在技术难度和合规风险

#### 成本估算

| 成本项 | 一次性成本 | 月度成本 |
|-------|-----------|---------|
| 开发人力 | ¥5,000-10,000 | - |
| 服务器 | - | ¥200-500 |
| AI API调用 | - | ¥300-1,000 |
| **合计** | **¥5,000-10,000** | **¥500-1,500** |

#### 适用场景
- ✅ 有专职技术人员支持
- ✅ 需求可能持续演进和调整
- ✅ 重视长期成本控制
- ✅ 需要与现有系统深度集成

1. 使用平台提供的RSS/网页监控模块

3. 通过Webhook或数据库连接器写入CRM
4. 配置通知流程（邮件/企业微信）

- ⚠️ **成本不确定**：按流程执行次数收费，难以预估
- ⚠️ **国内数据源支持差**：微信公众号、监管平台集成较弱

3. 集成外部工具：通过API/Webhook与爬虫和数据库对接
4. 人工审核界面：利用平台内置的对话界面或自建前端

#### 核心优势
- ✅ **AI原生**：专为AI应用设计，提取能力更强
- ✅ **知识库功能**：可持续积累和优化提取模式

- ⚠️ **私有部署门槛**：Dify部署需要一定技术能力
- ⚠️ **云平台限制**：Coze等云服务在功能和数据控制上有限制

| 爬虫模块开发 | ¥3,000-5,000 | - |

#### 适用场景
- AI提取是核心需求，数据采集可外包
- 希望持续优化AI提取效果
- 对数据安全有要求（选择私有部署）
- 团队愿意学习和探索新工具

### 5.2 数据库设计

#### 表结构设计

**1. 原始数据表 (raw_data)**

```sql
CREATE TABLE raw_data (
    id SERIAL PRIMARY KEY,
    source_type VARCHAR(50),           -- 来源类型：监管平台/公司官网/公众号/媒体
    source_url TEXT,                   -- 原文链接
    title TEXT,                        -- 文章标题
    content TEXT,                      -- 文章正文
    publish_date TIMESTAMP,            -- 发布时间
    crawl_time TIMESTAMP DEFAULT NOW(),-- 抓取时间
    status VARCHAR(20) DEFAULT 'pending', -- 处理状态：pending/processing/completed/failed
    company_name VARCHAR(100)          -- 相关公司（可为空，由AI提取）
);
```

**2. 提取信息表 (extracted_info)**

```sql
CREATE TABLE extracted_info (
    id SERIAL PRIMARY KEY,
    raw_data_id INTEGER REFERENCES raw_data(id),
    info_type VARCHAR(20),             -- 信息类型：personnel/equity
    company_name VARCHAR(100),         -- 公司名称
    person_name VARCHAR(50),           -- 人员姓名
    change_type VARCHAR(20),           -- 变动类型：任命/离职/调动/股权变更
    position VARCHAR(100),             -- 职位
    old_position VARCHAR(100),         -- 原职位（如有）
    effective_date DATE,               -- 生效日期
    equity_info JSONB,                 -- 股权信息（JSON格式）
    confidence FLOAT,                  -- AI置信度
    extract_time TIMESTAMP DEFAULT NOW(),
    review_status VARCHAR(20) DEFAULT 'pending', -- 审核状态
    reviewer VARCHAR(50),              -- 审核人
    review_time TIMESTAMP,
    review_note TEXT,                  -- 审核备注
    is_confirmed BOOLEAN DEFAULT FALSE,-- 是否已确认
    is_synced_to_crm BOOLEAN DEFAULT FALSE -- 是否已同步到CRM
);
```

**3. 监控公司列表 (monitored_companies)**

```sql
CREATE TABLE monitored_companies (
    id SERIAL PRIMARY KEY,
    company_name VARCHAR(100) UNIQUE,
    company_alias TEXT[],              -- 公司别名数组
    website_url TEXT,
    wechat_account VARCHAR(100),
    priority INTEGER DEFAULT 5,        -- 优先级：1-10
    is_active BOOLEAN DEFAULT TRUE,

```

### 5.3 AI提取Prompt设计

#### 人事变动提取Prompt示例

```
你是一个专业的金融信息提取助手。请从以下文本中提取人事变动信息。

**提取规则：**
1. 只提取信托公司的人事信息
2. 关注董事长、总经理、副总、部门总经理、总监及以上职位
3. 变动类型包括：任命、离职、调动
4. 如果文中未明确说明生效日期，可推测或标记为"未知"

**输出格式（JSON）：**
```json
{
  "company_name": "公司全称",

## 八、结论与建议

### 8.1 核心结论

1. **技术可行性高**：通过AI大模型进行信息提取是成熟的技术方案，经过POC验证可达到85%以上的准确率。

2. **投资回报明显**：相比人工监控，自动化系统可节省80%以上的人力成本，同时提升信息获取的及时性和全面性。

3. **推荐方案A**：Python轻量级方案在灵活性、可控性、成本效益方面最优，特别适合有技术团队支持的场景。

4. **分阶段实施风险可控**：通过POC验证 → MVP开发 → 试运行的渐进式路径，可有效控制风险。

### 8.2 实施建议

#### 短期行动（1个月内）

- **立即启动POC验证**：收集样本数据，测试AI提取效果，1-2周内完成可行性评估
- **确定技术方案**：基于POC结果，最终确定采用方案A、B或C
- **组建项目团队**：明确技术人员和研究人员的分工
- **申请预算**：根据选定方案编制详细预算

#### 中期规划（3-6个月）

- 完成MVP开发并上线试运行
- 建立持续优化机制：每周复盘误报和漏报，调整策略
- 逐步扩展监控范围：从信托公司扩展到其他金融机构
- 培训团队成员：确保研究人员熟练使用审核界面

#### 长期发展（6-12个月）

- **功能扩展**：增加股权变动、重大事件等更多监控维度
- **智能化升级**：基于积累的数据训练专用模型，进一步提升准确率
- **数据产品化**：将监控数据整合为定期报告或数据产品
- **行业拓展**：复制成功经验到银行、证券等其他金融领域

### 8.3 关键成功因素

2. **技术能力**：确保有Python开发能力的技术人员持续参与
3. **数据积累**：建立高质量的样本数据库，用于持续优化AI模型
4. **用户反馈**：研究人员积极反馈使用体验，帮助快速迭代
